General Document Structure

The following rules target elements that are (or should be) unique to each document. In some cases compound selectors are required in order to only select the specific element. Nevertheless, we can address them without any added class specificity.

html

scss 
html {
  font-size: var(--scale-root, 100%);
}

Document root. Also selectable as :root (which has a higher specificity).

body

scss 
body {
  color: var(--color-body);
  background-color: var(--color-body-negative);
  // Push down the footer.
  min-block-size: 100vh;
}

Content root.

main

scss 
main {
  padding-block: var(--spacing-sectioning);
}

The document’s main content. There can be only one (visible).

Related

Spec [external]

MDN [external]

:where(body, body > div) > header

scss 
:where(body, body > div) > header {
  @extend %clr-accented--inverted;
  padding-block: var(--spacing-sectioning);
}

The document’s main header. (There could be more than one. But that would not make much sense.)

:where(body, body > div) > footer

scss 
:where(body, body > div) > footer {
  // Clever sticky footer (https://css-tricks.com/a-clever-sticky-footer-technique/)
  position: sticky;
  inset-block-start: 100vh;

  // Spacing and color.
  @extend %clr-accented--inverted;
  padding-block: var(--spacing-sectioning);
}

The document’s main footer. (There could be more than one. But that would not make much sense.)
